1. 큐(Queue)
큐는 도착한 순서대로 데이터를 꺼내서 사용하는 선형적인 자료구조(Linear Data Structure)이다. 
큐는 흔히 FIFO(First In First Out) 으로 불리우는 자료구조로서 먼저 저장된 것이 먼저 꺼내지는 선입선출 구조를 가지고 있다.
만약 즉시 처리 하지않아도 되는일이라면, 큐에 차례로 넣어두고 순서대로 그 일들을 처리할 때 큐를 사용하곤한다.

2. 배열 큐
배열을 사용해 큐를 구현할 때는 고정배열 큐 보다는 원형배열 큐를 이용해서 구현한다.
고정된 배열의 크기에서 Front 인덱스를 읽다보면 금새 마지막 요소에 이르게된다. 이럴 경우 앞에 데이터를 추가할 수 있는 공간이 비워있는데 추가하지못해 비효율적이다.
이러한 문제때문에 원형배열을 이용해서 비어있는 공간을 효율적으로 사용할 수 있다.
원형 배열을 이용해 Queue 를 구성하면 데이터를 읽는 Front 포인터와 데이터를 쓰는 Rear 포인터를 이용해서 구현이 가능하다.

3. 연결 리스트 큐
연결 리스트를 이용해서 큐를 구현하면 배열을 이용하는 것보다 매우 간단하게 구현할 수 있다.
단일 연결리스트에서 Head, Tail를 사용하면 되며, 이는 배열의 Front(Head), Rear(Tail)과 같다.
단일 연결리스트는 큐에 사용한 데이터가 무한, 가변적일 때 사용하면 유용하다.
