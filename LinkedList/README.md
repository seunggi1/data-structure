 1. 연결리스트의 기초개념
  연길리스트(Linked List)는 각 노드가 데이터와 포인터를 가지고 있으면서 노드들이 한 줄로 쭉 연결되어 있는 방식으로 데이터를 저장하는 자료구조이다.
  노드가 다음 노드를 가르키고 있는 리스트를 단일 연결 리스트라 하고, 각 노드가 이전 노드와 다음 노드 모두를 가르키는 양방향 노드로 되어있는 것을 이중 연결 리스트라 한다.

 2. 단일 연결 리스트
  단일 연결리스트는 단방향으로 노드들을 연결한 간단한 자료구조이다.
  단일 연결리스트를 구현하기 위해서는 먼저 노드를 표현하는 노드 클래스와 이를 연결한 리스트인 링크드 리스트 클래스를 만들어야한다.
  노드들을 단방향으로 연결한 단일 연결 리스트 클래스는 리스트의 첫 노드를 가르키는 헤드 필드를 가르키게 되는데, 이 Head를 사용해서 전체리스트를 순차적으로 이동할 수 있다.

3. 이중 연결 리스트
 이중 연결 리스트는 단방향 연결 리스트를 개선한 것이다.
 각 노드들이 이전 노드와 다음 노드를 가지고있으며 양방향으로 탐색이 `가능한 자료 구조이다.

 4. 원형 연결 리스트
 원형 연결 리스트는 환형 연결 리스트라고도 하는데, 일반 연결 리스트에서 마지막 노드를 처음노드에 연결시켜 원형으로 만든 구조이다.
 원형 연결 리스트를 단방햐응로 연결한 것은 원형 단일 연결리스트(Singly Circular Linked List)라 하고, 양방향으로 연결한 것을 
 원형 이중 연결 리스트(Doubly Circular Linked List)라 한다.

 5. 원형 단일 연결 리스트의 변형
 Head가 마지막 노드를 가르키는 단일 연결 리스트이다. 이 리스트의 장점은 새 노드를 추가할 때 Head 바로뒤에 노드를 즉시 추가할 수 있고, 
 노드를 검색할 때는 Head 바로 다음 노드인 첫 노드로 이동하여 바로 일반 연결 리스트처럼 검색할 수 있다는 점이다.
 원형 이중 연결 리스트가 2개의 링크를 계속 관리 해야하는 반면, 이 리스트는 하나의 링크만 관리하고도  O(1)의 성능으로 새로운 노드로 추가가 가능하다.
